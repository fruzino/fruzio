Architecture

C Build
core

The Mint interpreter is a single C file (mint.c) compiled to a native executable. Variables, math, control flow, functions, lists, and the plugin registry are all handled in C. Features requiring heavy Python libraries are delegated to small helper scripts via system() calls.

# Windows (MinGW)
gcc -o mint.exe mint.c -lm

# Linux / macOS
gcc -o mint mint.c -lm

Helper Scripts
architecture

When Mint encounters an AI, browser, media, or dialog command, it shells out to the appropriate Python helper and prints its stdout to the terminal. Helper paths resolve relative to mint.exe at startup.

02

Setup

First-time Setup
windows

Run setup.bat once as Administrator. It will:

  1. Call mod.bat to install all Python dependencies.
  2. Detect the install directory and rewrite setup.reg with the correct absolute path.
  3. Import setup.reg to register .mint files with mint.exe.
  4. Launch mint.exe.

After setup, double-clicking any .mint file opens it directly in Mint.

Dependencies (mod.bat)
python

Edit mod.bat to add or remove Python packages. Safe to run repeatedly.

python -m pip install selenium playsound3 climage python-vlc ^
    openai litellm google-generativeai flask -q -U

To add a dependency for your own plugin, append it to this list.

Registry Association
windows

setup.bat rewrites setup.reg with the correct path before importing. If you move the Mint folder, re-run setup.bat.

Registry KeyValue
HKCR\.mintAssociates extension with MintFile.
HKCR\MintFile\DefaultIconPoints to mint.exe for the file icon.
HKCR\MintFile\shell\open\commandRuns mint.exe "%1" on open.
03

Plugin API

Plugin Structure
plugins

Each plugin is a subfolder inside plugins\ with exactly two files. Mint scans at startup and registers all commands automatically. No recompile needed.

plugins\
  └── MyPlugin\
      ├── Plugin.py    command handler
      └── Plugin.plug  manifest

Mint prints [plugin] Loaded: MyPlugin (N cmds) at startup for each registered plugin.

Plugin.plug Format
plugins

Plain text. Lines starting with # are comments.

# Plugin.plug -- manifest for MyPlugin
script = plugins/MyPlugin/Plugin.py
cmd    = mycommand
cmd    = anothercommand
FieldDescription
scriptPath to the Python handler, relative to mint.exe.
cmdA command to register. One line per command.
Plugin.py Template
plugins

Mint calls your script as:  python Plugin.py <command> "arg1" "arg2" ...

Print results to stdout. Errors to stderr go to mint_debug.txt.

import sys

def cmd_mycommand(args):
    name = args[0] if args else "World"
    print(f"Hello, {name}!")

def cmd_anothercommand(args):
    print(" ".join(args).upper())

DISPATCH = {
    "mycommand":      cmd_mycommand,
    "anothercommand": cmd_anothercommand,
}

if __name__ == "__main__":
    fn = DISPATCH.get(sys.argv[1])
    if fn:
        fn(sys.argv[2:])
    else:
        print(f"Unknown: {sys.argv[1]}", file=sys.stderr)
        sys.exit(1)

Use it in .mint exactly like a built-in:

mycommand(Alice)
anothercommand(hello from mint)

Developer Notes

py( code )
python

Executes raw Python via python -c. Useful for quick one-offs without writing a plugin.

py(print("Hello from Python"))
py(import os; print(os.getcwd()))
Multiline Blocks
syntax

Open a { on any line, close with } on another. Works for if, repeat, while, and func. The REPL shows ... inside an open block.

func greet {
    write(Hello!)
    write(Welcome to Mint.)
}

if(@score >= 75, {
    write(Passed!)
    write(Score: @score)
})
Semicolons
syntax

Multiple commands on one line separated by ;. Works at any nesting level.

x = 10; y = 20; z = calc(@x + @y); write(@z)
Capturing Plugin Output
tips

List commands auto-store results in @_. For plugin output, use sys() with shell redirection.

# List commands -- result auto-stored in @_
list.get(mylist, 0)
write(Got: @_)

# Plugin output -- redirect to file then load
sys(python plugins/strtools/Plugin.py upper hello > tmp.txt)
load(tmp.txt)

Contributor Workflow

Update Workflow
contributing
  1. Edit mint.c for core interpreter changes. Recompile: gcc -o mint.exe mint.c -lm
  2. Edit the appropriate helper script for Python-backed feature changes.
  3. For new commands in Python, create a plugin rather than editing core files.
  4. Update syntaxes/mint.tmLanguage.json in the VS Code extension for syntax highlighting.
  5. Update package.json or extension.js if needed.
  6. Run vsce package in the extension folder to rebuild the .vsix.
  7. Update commands.html with new command reference entries.
